TẠI SAO DÙNG ELASTICSEARCH THAY VÌ SQL?
================================================================================

1. TỐC ĐỘ - NHANH HƠN 10-50 LẦN ⚡
   
   SQL Server với LIKE '%keyword%':
   - 100,000 papers: 3-8 giây
   - 1,000,000 papers: 30-60 giây
   - Phải scan toàn bộ table
   - Không thể dùng index hiệu quả
   
   Elasticsearch:
   - 100,000 papers: 50-200ms (nhanh gấp 15-160 lần)
   - 1,000,000 papers: 200-500ms (nhanh gấp 60-300 lần)
   - Sử dụng inverted index (tối ưu cho search)
   - Scale tốt khi dữ liệu tăng

2. CHẤT LƯỢNG TÌM KIẾM - THÔNG MINH HƠN 🎯
   
   SQL LIKE operator:
   ❌ Case sensitive: "Machine Learning" ≠ "machine learning"
   ❌ Exact match only: "machine-learning" không match "machine learning"
   ❌ Không xử lý typo: "machne learning" không tìm được gì
   ❌ Sai thứ tự không match: "learning machine" không match
   ❌ Không có synonym: search "AI" không tìm được "Artificial Intelligence"
   
   Elasticsearch:
   ✅ Case insensitive: "Machine Learning" = "machine learning"
   ✅ Tokenization: tìm được "machine-learning", "machine_learning"
   ✅ Fuzzy search: "machne learning" vẫn tìm được "machine learning"
   ✅ Flexible matching: "learning machine" vẫn tìm được (score thấp hơn)
   ✅ Synonym support: search "AI" → tìm được "Artificial Intelligence"
   ✅ Stemming: "running" = "run" = "runs" = "ran"
   ✅ Stop words: bỏ qua "a", "the", "is" để focus vào từ quan trọng

3. RELEVANCE RANKING - TỰ ĐỘNG SẮP XẾP 📊
   
   SQL:
   ❌ Không biết paper nào "liên quan nhất"
   ❌ Kết quả trả về theo thứ tự ngẫu nhiên hoặc theo date
   ❌ Phải tự viết logic ranking phức tạp
   
   Elasticsearch:
   ✅ TF-IDF scoring: từ khóa hiếm → điểm cao hơn
   ✅ Field boosting: từ khóa trong title quan trọng hơn abstract (x2)
   ✅ Term frequency: từ khóa xuất hiện nhiều lần → relevance cao hơn
   ✅ Document length: normalize theo độ dài document
   ✅ Kết quả tự động sort theo mức độ liên quan
   
   Ví dụ search "machine learning":
   - Paper 1: Title có "Machine Learning" → Score: 8.5 → Rank #1
   - Paper 2: Abstract có "machine learning" 5 lần → Score: 6.2 → Rank #2
   - Paper 3: Abstract có "machine learning" 1 lần → Score: 3.1 → Rank #3

4. HIGHLIGHTING - HIGHLIGHT TỰ ĐỘNG 🔦
   
   SQL:
   ❌ Trả về full text, không biết từ khóa ở đâu
   ❌ Phải tự viết code regex để highlight
   ❌ Frontend phải xử lý logic phức tạp
   
   Elasticsearch:
   ✅ Tự động wrap từ khóa với <em> tag
   ✅ Chỉ trả về fragment có từ khóa (tiết kiệm bandwidth)
   ✅ Configurable: số lượng fragments, độ dài fragment
   ✅ Frontend chỉ cần render HTML
   
   Response example:
   {
     "title": "Deep Learning Networks",
     "highlight": {
       "title": ["Deep <em>Learning</em> Networks"],
       "abstract": ["neural networks for <em>learning</em>..."]
     }
   }

5. TÍNH NĂNG NÂNG CAO 🚀
   
   ✅ Multi-field search: Tìm đồng thời trong title, abstract, keywords
   ✅ Boolean operators: "machine learning" AND "neural networks"
   ✅ Range queries: publicationYear: 2020-2024
   ✅ Aggregations: "Có bao nhiêu papers mỗi năm?"
   ✅ Geo search: Tìm papers theo location (nếu có)
   ✅ More like this: "Tìm papers tương tự paper này"
   ✅ Autocomplete: Suggest khi user đang gõ
   ✅ Language analyzers: Tự động xử lý 30+ ngôn ngữ

6. CACHING VỚI REDIS - NHANH HƠN NỮA ⚡
   
   Architecture:
   User search → Redis cache (5-20ms) → Elasticsearch (100-200ms) → SQL (3000ms+)
                    ↑ Cache hit          ↑ Cache miss
   
   ✅ First search: 150ms (Elasticsearch)
   ✅ Subsequent searches: 10ms (Redis) - Nhanh gấp 15 lần
   ✅ Cache TTL: 15 phút (configurable)
   ✅ Automatic cache key generation (MD5 hash of query params)
   ✅ Save server resources và database load

7. SCALABILITY - DỄ DÀNG MỞ RỘNG 📈
   
   SQL Server:
   ❌ Vertical scaling only (tăng RAM, CPU)
   ❌ Full-text search index rất nặng
   ❌ Query phức tạp → lock database
   
   Elasticsearch:
   ✅ Horizontal scaling: Thêm nodes vào cluster
   ✅ Automatic sharding: Data tự động phân tán
   ✅ Replication: High availability
   ✅ No locking: Reads không block writes

8. USER EXPERIENCE - TRẢI NGHIỆM TốT HƠN 😊
   
   ✅ Instant search: Kết quả hiện ngay (< 200ms)
   ✅ Typo-tolerant: User gõ sai vẫn tìm được
   ✅ Relevant results: Paper quan trọng nhất ở trên đầu
   ✅ Highlighted keywords: User thấy ngay tại sao paper được tìm thấy
   ✅ Fast pagination: Chuyển trang nhanh
   ✅ Filters: Dễ dàng lọc theo năm, language, open access

9. SO SÁNH THỰC TẾ
   
   Tình huống: 100,000 papers, user search "deep learning neural networks"
   
   ┌─────────────────┬──────────────┬─────────────────┐
   │ Metric          │ SQL Server   │ Elasticsearch   │
   ├─────────────────┼──────────────┼─────────────────┤
   │ Query time      │ 3,500ms      │ 150ms (23x)     │
   │ Relevance       │ Random       │ Sorted by score │
   │ Highlighting    │ No           │ Auto            │
   │ Typo handling   │ No           │ Yes (fuzzy)     │
   │ Multi-language  │ Manual code  │ Built-in        │
   │ Scale to 1M     │ 30,000ms     │ 500ms (60x)     │
   │ Cached result   │ N/A          │ 10ms (Redis)    │
   └─────────────────┴──────────────┴─────────────────┘

10. KẾT LUẬN
    
    Elasticsearch KHÔNG THAY THẾ SQL Server!
    
    ✅ SQL Server: Storage, CRUD, Transactions, Joins, ACID
    ✅ Elasticsearch: Search, Analytics, Aggregations
    ✅ Redis: Caching, Session storage
    
    Kiến trúc tối ưu:
    [User] → [API]
                ↓
         [Redis Cache] (15 phút)
                ↓
         [Elasticsearch] (Search)
                ↓
         [SQL Server] (Source of truth)
    
    Workflow:
    1. User tạo/update paper → Save vào SQL Server
    2. Trigger reindex → Sync vào Elasticsearch
    3. User search → Query Elasticsearch → Cache Redis
    4. Kết quả nhanh, chính xác, relevant!

1. SEARCH API - GET /api/search/papers
================================================================================

MÔ TẢ:
Tìm kiếm full-text papers với Elasticsearch, kết quả được cache trong Redis 15 phút.

AUTHENTICATION: Required (Bearer Token)

QUERY PARAMETERS:
┌─────────────┬─────────┬──────────┬──────────────────────────────────────┐
│ Parameter   │ Type    │ Required │ Description                          │
├─────────────┼─────────┼──────────┼──────────────────────────────────────┤
│ q           │ string  │ No       │ Từ khóa tìm kiếm (title, abstract)   │
│ page        │ int     │ No       │ Số trang (default: 1)                │
│ size        │ int     │ No       │ Số kết quả/trang (default: 10)       │
│ from        │ int     │ No       │ Năm bắt đầu (VD: 2020)               │
│ to          │ int     │ No       │ Năm kết thúc (VD: 2024)              │
│ language    │ string  │ No       │ Ngôn ngữ (VD: en, vi)                │
│ isOpenAccess│ bool    │ No       │ Lọc open access (true/false)         │
└─────────────┴─────────┴──────────┴──────────────────────────────────────┘

REQUEST EXAMPLE (cURL):
```bash
curl -X GET "http://localhost:8080/api/search/papers?q=machine%20learning&page=1&size=10&from=2020&to=2024&isOpenAccess=true" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

REQUEST EXAMPLE (JavaScript/Axios):
```javascript
const response = await axios.get('/api/search/papers', {
  params: {
    q: 'machine learning',
    page: 1,
    size: 10,
    from: 2020,
    to: 2024,
    language: 'en',
    isOpenAccess: true
  },
  headers: {
    'Authorization': `Bearer ${token}`
  }
});
```

REQUEST EXAMPLE (C# HttpClient):
```csharp
var query = new Dictionary<string, string>
{
    { "q", "machine learning" },
    { "page", "1" },
    { "size", "10" },
    { "from", "2020" },
    { "to", "2024" },
    { "isOpenAccess", "true" }
};
var queryString = await new FormUrlEncodedContent(query).ReadAsStringAsync();

```

RESPONSE (200 OK):
```json
{
  "success": true,
  "data": {
    "total": 150,
    "page": 1,
    "size": 10,
    "results": [
      {
        "paperId": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
        "title": "Deep Machine Learning Techniques",
        "abstract": "This paper explores various machine learning algorithms...",
        "publicationYear": 2024,
        "citedByCount": 42,
        "highlight": {
          "title": ["Deep <em>Machine Learning</em> Techniques"],
          "abstract": ["explores various <em>machine learning</em> algorithms"]
        }
      },
      {
        "paperId": "7fb95f64-6828-5673-c4gd-3d074g77bgb7",
        "title": "Advanced Neural Networks",
        "abstract": "A comprehensive study on neural network architectures...",
        "publicationYear": 2023,
        "citedByCount": 28,
        "highlight": {
          "title": ["Advanced Neural Networks"],
          "abstract": ["study on neural network architectures"]
        }
      }
    ]
  }

```

RESPONSE FIELDS:
- total: Tổng số kết quả tìm được
- page: Trang hiện tại
- size: Số kết quả mỗi trang
- results[]: Mảng các paper tìm được
  - paperId: ID của paper
  - title: Tiêu đề
  - abstract: Tóm tắt
  - publicationYear: Năm xuất bản
  - citedByCount: Số lần được trích dẫn
  - highlight: Phần text được highlight với từ khóa tìm kiếm
    - title[]: Tiêu đề có highlight (HTML với <em> tag)
    - abstract[]: Đoạn abstract có highlight

ERROR RESPONSES:
- 401 Unauthorized: Không có hoặc sai token
- 400 Bad Request: Sai format parameters

CACHE BEHAVIOR:
- Kết quả được cache trong Redis 15 phút
- Cache key dựa trên MD5 hash của tất cả parameters
- Request giống nhau sẽ trả về từ cache (nhanh hơn)

2. ADMIN API - POST /api/admin/reindex
================================================================================

MÔ TẢ:
Đồng bộ TẤT CẢ papers từ SQL Server vào Elasticsearch. Xóa index cũ và tạo lại.

AUTHENTICATION: Required (Bearer Token - Admin role khuyến nghị)

KHI NÀO SỬ DỤNG:
- Lần đầu tiên setup hệ thống
- Sau khi import nhiều papers mới vào database
- Khi Elasticsearch bị lỗi hoặc mất đồng bộ
- Sau khi thay đổi cấu trúc PaperDocument

CẢNH BÁO:
⚠️ API này có thể mất nhiều thời gian nếu có nhiều papers (vài phút đến vài chục phút)
⚠️ Không nên gọi thường xuyên vì tốn tài nguyên
⚠️ Trong quá trình reindex, search có thể trả về kết quả không đầy đủ

REQUEST EXAMPLE (cURL):
```bash
curl -X POST "http://localhost:8080/api/admin/reindex" \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN"
```

REQUEST EXAMPLE (JavaScript/Axios):
```javascript
const response = await axios.post('/api/admin/reindex', {}, {
  headers: {
    'Authorization': `Bearer ${adminToken}`
  }
});
```

REQUEST EXAMPLE (C# HttpClient):
```csharp
var response = await httpClient.PostAsync("/api/admin/reindex", null);
```

RESPONSE (200 OK):
```json
{
  "success": true,
  "data": {
    "success": true,
    "message": "Reindex executed directly (Hangfire disabled)"
  }
}
```

QUY TRÌNH THỰC HIỆN:
1. Xóa index "papers" cũ (nếu có)
2. Tạo index "papers" mới với mapping
3. Query TẤT CẢ papers từ SQL Server (SELECT * FROM core.papers)
4. Convert sang PaperDocument
5. Bulk insert vào Elasticsearch (batch)
6. Trả về kết quả

LOG EXAMPLE:
```
[12:30:45] Starting reindex of all papers
[12:30:46] Index papers deleted successfully
[12:30:47] Index papers created successfully
[12:30:48] Retrieved 10000 papers from database
[12:31:15] Bulk indexing completed
[12:31:15] Reindex completed. Total papers indexed: 10000
```

- Khi muốn force tất cả user search lại từ Elasticsearch

REQUEST EXAMPLE (cURL):
```bash
curl -X POST "http://localhost:8080/api/admin/cache/clear" \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN"
```

REQUEST EXAMPLE (JavaScript/Axios):
```javascript
const response = await axios.post('/api/admin/cache/clear', {}, {
  headers: {
    'Authorization': `Bearer ${adminToken}`
  }
});
```

RESPONSE (200 OK):
```json
{
  "success": true,
  "data": {
    "cleared": true,
    "message": "Cache cleared successfully"
  }

```

- Xóa tất cả keys có pattern "search:papers:*" trong Redis
- Search request tiếp theo sẽ query từ Elasticsearch (chậm hơn)
- Cache sẽ được build lại dần khi user search

1. SEARCH COMPONENT (React Example)

```javascript
import { useState, useEffect } from 'react';
import axios from 'axios';

const SearchPapers = () => {
  const [query, setQuery] = useState('');
  const [results, setResults] = useState(null);
  const [loading, setLoading] = useState(false);
  const [filters, setFilters] = useState({
    page: 1,
    size: 10,
    from: null,
    to: null,
    language: '',
    isOpenAccess: null
  });

  const search = async () => {
    setLoading(true);
    try {
      const token = localStorage.getItem('jwt_token');
      const response = await axios.get('/api/search/papers', {
        params: { q: query, ...filters },
        headers: { 'Authorization': `Bearer ${token}` }
      });
      setResults(response.data.data);
    } catch (error) {
      console.error('Search failed:', error);
    } finally {
      setLoading(false);
    }
  };

  return (
    <div>
      <input 
        value={query} 
        onChange={(e) => setQuery(e.target.value)}
        placeholder="Search papers..."
      />
      <button onClick={search}>Search</button>

      {loading && <p>Searching...</p>}

      {results && (
        <div>
          <p>Found {results.total} papers</p>
          {results.results.map(paper => (
            <div key={paper.paperId}>
              <h3 dangerouslySetInnerHTML={{ 
                __html: paper.highlight.title[0] || paper.title 
              }} />
              <p>{paper.publicationYear} • Citations: {paper.citedByCount}</p>
              <p dangerouslySetInnerHTML={{
                __html: paper.highlight.abstract[0] || paper.abstract
              }} />
            </div>
          ))}
        </div>
      )}
    </div>
  );
};
```

```javascript
import axios from 'axios';

const AdminPanel = () => {

      <button onClick={clearCache}>Clear Cache</button>
```

PERFORMANCE NOTES
================================================================================

ELASTICSEARCH:
- First search (cache miss): ~100-500ms
- Subsequent search (cache hit): ~5-20ms
- Reindex 10,000 papers: ~1-3 phút
- Reindex 100,000 papers: ~10-30 phút

REDIS CACHE:
- TTL: 15 phút (configurable)
- Memory usage: ~1KB per cached search result
- Eviction: Automatic sau 15 phút

RECOMMENDATIONS:
- Limit page size tối đa 100 results
- Sử dụng pagination thay vì load all
- Debounce search input (300-500ms)
- Show loading indicator khi search
- Cache search results ở client side (session storage)


================================================================================
TROUBLESHOOTING
================================================================================

PROBLEM: Search trả về 0 results
SOLUTION: 
1. Kiểm tra Elasticsearch có chạy: curl http://localhost:9200
2. Kiểm tra index tồn tại: curl http://localhost:9200/papers
3. Reindex: POST /api/admin/reindex

PROBLEM: Search chậm
SOLUTION:
1. Kiểm tra Redis có chạy: redis-cli ping
2. Kiểm tra cache hit rate trong logs
3. Tăng Redis cache TTL nếu data ít thay đổi

PROBLEM: Kết quả search không cập nhật
SOLUTION:
1. Clear cache: POST /api/admin/cache/clear
2. Reindex: POST /api/admin/reindex

PROBLEM: Reindex bị timeout
SOLUTION:
1. Tăng timeout trong HTTP client
2. Chạy reindex off-peak hours
3. Enable Hangfire để chạy background (hiện đang disabled)
